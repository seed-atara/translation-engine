# Lingua — v0.5 Overview

**AI-native translation engine for marketing agencies**
Built by Seed Studios · May 2026

---

## What is this?

Lingua replaces the traditional translation agency workflow — brief a translator, wait, receive a Word doc, chase edits — with a five-stage Claude AI pipeline that runs in under 60 seconds and gets smarter over time.

The target user is a marketing professional managing multi-market campaigns. They paste copy, choose a language, and get back a translation that reads as native — with cultural context, safety checks, and a confidence score. A human editor only touches the work that genuinely needs them.

**Live URL:** https://translation-engine-production.up.railway.app

---

## The Pipeline

Every translation passes through five Claude agents in sequence:

```
Source Copy
    │
    ▼
① Idiom Agent          (Claude Sonnet 4.6)
  Identifies idioms, slang, metaphors
  Normalises them to literal meaning before translation
    │
    ▼
② Cultural Agent       (Claude Sonnet 4.6)
  Researches target locale: trends, register, taboos, glossary
  Outputs structured cultural brief for the translator
    │
    ▼
③ Translation Agent    (Claude Opus 4.7 / Sonnet 4.6)
  Translates with full cultural context + client tone of voice injected
  Scores confidence per segment
  Uses editor-approved corrections as few-shot examples
    │
    ▼
④ Safety Agent         (Claude Haiku 4.5)
  Flags content inappropriate for professional brands
  Replaces problematic phrases, notes severity
    │
    ▼
⑤ QA Agent             (Claude Haiku 4.5)
  Scores overall translation quality (0–1)
  Writes reviewer notes prioritising where human attention is needed
  Sets humanReviewRequired flag
```

Agents communicate via typed JSON handoffs only — no conversation history accumulates, so token cost stays flat regardless of copy length.

## What's Built (v0.5)

### Core pipeline ✅
- All five agents implemented and running in production
- Async generator pipeline streams events to the UI in real time
- Schema-resilient JSON parsing — handles all Claude output format variants
- Model routing: Opus 4.7 for technical docs, Sonnet 4.6 for standard copy, Haiku 4.5 for verification
- Prompt caching on client system prompts (reduces cost on repeat jobs)

### Translation workspace ✅
- Live agent progress tracker — see each stage tick off as it completes
- Expandable Idiom panel: every idiom found with original → normalised mapping and type tag
- Expandable Cultural panel: register, avoid phrases, glossary terms
- Translation output with confidence score colour-coding
- Cultural adaptations, safety flags, and reviewer notes (all expandable)

### Multi-tenant data layer ✅
- Agency → Client → TranslationJob hierarchy
- Per-client tone of voice and audience profile (JSON)
- Client document storage via Anthropic Files API (brand guidelines injected into translation)
- Translation memory: editor corrections stored as weighted records, injected as few-shot examples into future jobs

### Infrastructure ✅
- Deployed on Railway (app + Postgres, co-located)
- SSE streaming endpoint — no polling, no websockets
- Health check endpoint, restart-on-failure policy
- Seeded demo agency + client data

### Dashboard & navigation ✅
- Dashboard with stats (total jobs, clients, avg confidence, pending review)
- Client cards with job counts
- Recent jobs table
- Analytics page (language pair breakdown, status distribution)
- Add client flow (name, domain, target languages, tone of voice)
- Client detail page with translation history

## What's Not Built Yet

### Phase 3 — Asana Integration
The pipeline creates jobs and stores results in Postgres, but they don't yet push to Asana automatically.

- [ ] Asana OAuth flow (token exchange, stored per client)
- [ ] Custom field creation on Asana projects (source lang, target lang, confidence score)
- [ ] "Push to Asana" button in workspace
- [ ] Webhook receiver: Asana story comments parsed as corrections → translation memory

**Why it matters:** This is the editorial handoff — the moment the system hands a job to a human and learns from their response. Without it, the learning loop is manual.

### Phase 4 — In-App Review Interface
- [ ] `/review/[jobId]` — side-by-side diff of source and translation
- [ ] Inline segment editing with correction type selector
- [ ] Approve / reject job actions
- [ ] Editor email notifications

**Why it matters:** Clients without Asana need a way to review and feed corrections back to the system.

### Phase 5 — Analytics (partial)
The analytics page exists but shows aggregate counts only.

- [ ] Confidence score trend over time (per client, per language)
- [ ] `humanReviewRequired` rate declining over time — the core proof-of-learning metric
- [ ] Translation memory size growth per client
- [ ] Cost estimator (tokens per job, rolling monthly cost)
- [ ] CSV export

### Phase 6 — Production Hardening
- [ ] Authentication (NextAuth is installed; login UI is not wired)
- [ ] Rate limiting on the translate endpoint
- [ ] Job queue for parallel batch requests
- [ ] Error boundaries and toast notifications throughout UI
- [ ] Structured logging + Railway Metrics dashboard
- [ ] Staging environment

### Other gaps
- Document upload UI (the Files API integration exists in the backend; there's no upload screen yet)
- Saving completed translations to the database (the job runs, the result appears on screen, but it's not persisted to `TranslationJob` table yet)
- Per-segment confidence display in the output panel (scores are computed, not surfaced in the UI)
- DOCX / PPTX file input — currently paste-only

## Showing the Tool

**Demo script:**

1. Open https://translation-engine-production.up.railway.app
2. Click **Demo Brand** → **New Translation**
3. Set language to **Spanish**, content type **Marketing Copy**
4. Paste the text below and click **Translate**
5. Watch the five agents tick off — click into the **Idioms** and **Cultural** expandable panels
6. The translation appears with a confidence score and reviewer notes

**Demo copy (idiom-heavy UK marketing):**
```
We've hit the ground running with our latest product launch, and the results speak for themselves. Our team has been burning the midnight oil to ensure every detail is spot on — and it's paid off in spades.

This isn't just another flash in the pan. We're in it for the long haul, and we're pulling out all the stops to make sure our customers get the best bang for their buck.

The ball is firmly in our court now. We've got the wind in our sails, a world-class product, and a team that goes the extra mile every single day. The proof is in the pudding — our NPS score has gone through the roof.

Join us. Let's hit the ground running together.
```

The idiom agent strips all eight idioms and hands normalised copy to the translation agent. The cultural agent researches the Spanish urban 25–45 demographic. The result should read as native Castilian Spanish, not a literal translation.
